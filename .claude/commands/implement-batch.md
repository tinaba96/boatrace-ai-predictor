# Linear チケット一括実装

引数: $ARGUMENTS (カンマ区切りのLinearチケットID。例: BOAT-123,BOAT-124,BOAT-125)

## 手順

### 1. チケット情報の一括取得
Linear MCP を使って、指定された全チケットの詳細情報を取得してください。

### 2. 依存関係の分析
チケット間の依存関係を確認し、以下を判断してください：
- 独立して並列実装できるチケット
- 順序に依存関係があるチケット

### 3. 並列実装（Agent Teams）
独立したチケットごとに、**別々のエージェントを並列で**起動してください。
各エージェントは worktree isolation モードで実行し、それぞれ：

1. `feature/{チケットID}-{概要}` ブランチを作成
2. チケットの要件に基づいて実装
3. ビルド検証
4. コミット（メッセージにチケットIDを含める）

### 4. 結果レポート
全エージェントの完了後、以下をレポートしてください：
- 各チケットの実装結果（成功/失敗）
- 作成されたブランチ一覧
- 次のステップ（PR作成、手動確認が必要な箇所等）

### 5. Linear ステータス更新
成功したチケットのステータスを Linear MCP で「進行中」に更新してください。

## 注意事項
- 最大3チケットまでの並列実装を推奨（リソース制約）
- コンフリクトの可能性がある場合はユーザーに警告
- 各チケットの実装は独立した worktree で行うため、コンフリクトは後で解決可能
