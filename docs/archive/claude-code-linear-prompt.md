# Claude Code用 - Linear統合プロンプト

このプロンプトをClaude Codeに渡すと、実装中に自動的にLinearチケットを作成・更新します。

## 📋 使用方法

```bash
claude-code --prompt .claude/claude-code-linear-prompt.md "機能を実装してください"
```

または、プロンプトの内容を直接指定：

```bash
claude-code "以下の機能を実装してください。実装開始時にLinearチケットを作成し、完了時に更新してください。"
```

---

## 🎯 Linear統合の指示

実装を開始する前に、以下の手順を必ず実行してください：

### 1. 実装開始時

1. **実装する機能を明確にする**
   - 機能名、目的、実装内容を整理

2. **Linearチケットを作成**
   - タイトル: `[機能名]` または `[バグ修正: バグ名]`
   - 説明: 実装内容の詳細、関連ドキュメントへの参照
   - コマンド: `npm run linear:create "[タイトル]" "[説明]"`

3. **チケットを「進行中」に更新**
   - 作成されたチケットIDを取得
   - コマンド: `npm run linear:update [チケットID] "進行中" "実装を開始"`

### 2. 実装中

- **重要なマイルストーンに到達したら、コメントを追加**
  - コマンド: `npm run linear:comment [チケットID] "進捗: [内容]"`

- **気づきや改善点があれば、新しいチケットを作成**
  - コマンド: `npm run linear:create "[気づきのタイトル]" "[詳細]"`

### 3. 実装完了時

1. **チケットを「完了」に更新**
   - コマンド: `npm run linear:update [チケットID] "完了" "実装完了"`

2. **実装内容の概要をコメントに記録**
   - コマンド: `npm run linear:comment [チケットID] "実装完了: [実装内容の概要]"`

## 📝 実装例

### 例1: 機能実装

```
以下の機能を実装してください。

【機能名】
予測機能の精度向上

【実装内容】
- 過去データの分析機能を追加
- 予測アルゴリズムの改善

【Linear統合】
1. Linearチケットを作成（タイトル: 予測機能の精度向上）
2. チケットを「進行中」に更新
3. 実装を開始
4. 実装完了時にチケットを「完了」に更新
```

### 例2: バグ修正

```
以下のバグを修正してください。

【バグの説明】
ログイン機能でエラーが発生する

【修正内容】
- 認証トークンの処理を修正
- エラーハンドリングを改善

【Linear統合】
1. Linearチケットを作成（タイトル: バグ修正: ログイン機能のエラー）
2. チケットを「進行中」に更新
3. 原因を調査し、コメントに記録
4. 修正を実施
5. テストを実施
6. チケットを「完了」に更新
```

## 🔧 コマンドリファレンス

### タスクを作成
```bash
npm run linear:create "タイトル" "説明"
```

### タスクを更新
```bash
npm run linear:update [チケットID] "状態" "コメント"
```

### コメントを追加
```bash
npm run linear:comment [チケットID] "コメント"
```

### タスク一覧を表示
```bash
npm run linear:list
```

### タスクの詳細を表示
```bash
npm run linear:get [チケットID]
```

## ⚠️ 重要な注意事項

1. **必ず実装開始時にLinearチケットを作成する**
   - 実装を開始する前に、必ずチケットを作成してください

2. **チケットIDを記録する**
   - 作成されたチケットIDは、実装中に使用するため記録してください

3. **進捗を定期的に記録する**
   - 重要なマイルストーンに到達したら、コメントで記録してください

4. **実装完了時に必ず更新する**
   - 実装が完了したら、必ずチケットを「完了」に更新してください

## 🚀 実装フロー

```
1. 実装タスクを理解
   ↓
2. Linearチケットを作成
   ↓
3. チケットを「進行中」に更新
   ↓
4. 実装を開始
   ↓
5. 進捗をコメントで記録（必要に応じて）
   ↓
6. 実装完了
   ↓
7. チケットを「完了」に更新
   ↓
8. 実装内容の概要をコメントに記録
```
