# プロジェクト課題一覧

## 1. 開発フローが未定義

### 現状の問題
- ローカル環境と本番環境の使い分けが不明確
- テストから本番pushまでの流れが定義されていない
- 直接masterにpushしている

### あるべき姿
```
[ローカル開発] → [プレビュー確認] → [本番デプロイ]
     ↓                ↓                  ↓
  npm run dev    Vercel Preview      master push
  localhost:5173  xxx.vercel.app     boatai.net
```

### 解決策（案）
1. **ブランチ戦略を定義**
   - `master`: 本番環境（プロテクト）
   - `develop`: 開発用（Vercel Previewで確認）
   - `feature/*`: 機能開発用

2. **環境変数の整理**
   - `.env.example` にすべての必要な変数を記載
   - ローカル用 / 本番用の違いを明記

3. **デプロイフローのドキュメント化**
   - `docs/setup/deployment-flow.md` を作成

### 関連ファイル
- `vercel.json`
- `.env.example`
- `.env.local`

## 2. 分析PDCAの体制が未整備

### 現状の問題
- 会場別モデル開発のための分析手法が確立されていない
- 分析結果をどこに蓄積し、どう活用するかが不明確
- 「仮説→検証→改善」のサイクルを回す仕組みがない
- 過去の分析結果を振り返る手段がない

### あるべき姿
```
[仮説立案] → [データ分析] → [ルール発見] → [検証] → [本番適用] → [結果確認]
     ↑                                                              ↓
     └──────────────────── フィードバック ←─────────────────────────┘
```

### 必要なもの
1. **分析テンプレート**: 毎回同じ形式で分析を記録
2. **結果の蓄積場所**: 会場ごとの分析履歴
3. **検証の仕組み**: 発見したルールの信頼性を確認
4. **振り返りの仕組み**: 過去の分析と結果を比較

### 解決策（案）

#### ディレクトリ構造
```
data/
├── analysis/
│   ├── venue-01/                    # 桐生
│   │   ├── 2025-01-13-hypothesis.json   # 仮説
│   │   ├── 2025-01-13-analysis.json     # 分析結果
│   │   ├── 2025-01-13-rules.json        # 発見したルール
│   │   └── README.md                    # 会場の特徴まとめ
│   ├── venue-03/                    # 江戸川
│   │   └── ...
│   └── summary/                     # 全体サマリー
│       ├── profitable-rules.json   # 回収率100%超えルール一覧
│       └── failed-hypotheses.json  # 失敗した仮説（学習用）
```

#### 分析ワークフロー
1. **仮説**: 「江戸川で1号艇A1+モーター40%以上なら勝率高い？」
2. **スクリプト実行**: `node scripts/analysis/test-hypothesis.js --venue=03 --condition="..."`
3. **結果保存**: 自動で `data/analysis/venue-03/` に保存
4. **検証**: 別期間のデータで再検証
5. **適用判断**: 信頼性が高ければモデルに組み込み

#### 必要なスクリプト
- `scripts/analysis/test-hypothesis.js` - 仮説検証用
- `scripts/analysis/find-conditions.js` - 条件探索用
- `scripts/analysis/verify-rule.js` - ルール検証用
- `scripts/analysis/summarize-results.js` - 結果サマリー生成

### 関連ファイル
- `scripts/analysis/`
- `data/analysis/`
- `.claude/CLAUDE.md`（分析ワークフローセクション）

## 3. サブエージェントの使い所が不明確

### 現状の問題
- Claude Codeのサブエージェント（Task tool）をいつ使うべきか判断基準がない
- 分析タスクに使うべきか悩む

### 整理

| 場面 | サブエージェント | 理由 |
|------|-----------------|------|
| 24会場の初期データ収集 | ○ | 並列で効率化できる |
| 複数条件の同時テスト | ○ | 並列で効率化できる |
| 1会場ずつ分析→相談 | ✕ | 対話的に進める必要がある |
| コードベース探索 | ○ | Explore agentが適切 |
| 長時間かかる処理 | ○ | バックグラウンド実行 |

### 結論（暫定）
- **対話的なタスク**（結果を見て相談、PDCAを回す）→ サブエージェント不使用
- **並列化できる独立したタスク** → サブエージェント使用
- 運用しながらルール化を検討
