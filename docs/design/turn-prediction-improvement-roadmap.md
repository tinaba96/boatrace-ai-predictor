# 展開予測（Turn Prediction）改善ロードマップ

**目標:** 72.5% → 80%+ の精度向上  
**分析期間:** 2026-03-10 ～ 2026-05-17 (1000レース)  
**現状:** UI表示順の上位3技術のいずれかが実績1着コースと一致する確率

---

## 🎯 根本原因分析

### 確認された構造的バイアス
1. **逃げ（escape）への過度な依存**
   - 予測数: 977/1000 (97.7%)
   - 実際発生: 515/1000 (51.5%)
   - 1コースのみの逃げに限定 → 1コース予測が過度に高い

2. **外郭コース（4-6コース）の予測失敗**
   - 実績: 4コース 101, 5コース 54, 6コース 35 = 190レース (19%)
   - 的中: 13, 3, 0 = 16レース (8.4%)
   - 不的中率: 91.6%

3. **まくり系技術の予測不足**
   - 実際の「まくり」: 182レース (18.2%)
   - 予測「まくり」: 17レース (1.7%)
   - 実際の「まくり差し」: 122レース (12.2%)
   - 予測「まくり差し」: 17レース (1.7%)

4. **決まり手別精度の大きなばらつき**
   | 決まり手 | 的中率 | サンプル |
   |---------|------|--------|
   | 逃げ | 99.42% | 515 |
   | 差し | 68.29% | 123 |
   | 抜き | 54.72% | 53 |
   | まくり | 42.86% | 182 |
   | まくり差し | **18.03%** | 122 |
   | 恵まれ | 0% | 4 |

5. **会場別精度の大きなばらつき**
   - 高精度（大村62-96%）: 1コース勝率が高い会場
   - 低精度（平和島57%, 桐生56%）: 1コース勝率が低い会場（実績47-48% vs 予想52-55%）

### コード分析（scripts/lib/turnPrediction.js）

**問題1: 逃げは1コースのみに制限**
```javascript
// Line 265-266: アーキテクチャ上の制約（正しい）
if (t === "nige" && courseInt !== 1) continue;
```
→ 逃げが予測されると必ず1コース。この制約自体は問題ではなく、**逃げの過度な予測**が問題。

**問題2: 逃げ確率の系統的な過小評価（最重要・要キャリブレーション）**
```javascript
// Line 29: SOFTMAX_TEMP=1.5、Line 33-34: makuri/makurizashi フロア
const SOFTMAX_TEMP = 1.5;
const MAKURI_PROB_FLOOR = 0.015;
const MAKURIZASHI_PROB_FLOOR = 0.012;
```
→ distribution.nige の予測値（平均40.5%）が実測（53.1%）より約14pt低い。これは「逃げパターン数」とは別問題で、確率値そのもののキャリブレーションずれ。逃げ rawProb を実測へ寄せる補正が必要。**SOFTMAX_TEMP を下げる・フロアを一律上げる方向は逃げを更に押し下げるため不可。**

**問題3: upsetFactor の逃げ抑制が過大**
```javascript
// Line 311-317: 最大25%の逃げ抑制
if (t === "nige") {
  rawProb *= 1 - upsetFactor * 0.25; // 実測効果の約5倍 — 過大
}
```
→ 風速7m+でも実測の逃げ率低下は0-2m比 -4.8pt のみ。最大25%抑制は実測効果の約5倍。係数は**強化ではなく1/3〜1/5へ縮小**すべき。

**問題4: 会場別パラメータの不完全な反映**
```javascript
// Line 223-230: 会場補正は 1コースの勝率比のみ
const venue1Rate = VENUE_1COURSE_WIN_RATE[venueCode];
```
→ 会場の「荒れやすさ」の全体的な特性が十分に反映されていない

## 📊 会場別特性分析

### 高精度会場の成功要因（81-96%）
**会場:** 下関(96%), 徳山(83%), 浜名湖(82%)

✅ **共通特性:**
1. 1コース実績勝率が高い (55-91%)
2. パラメータ想定値と実績が一致している
3. 逃げ（1コース）での決着が支配的 (66-88%)
4. 無風～弱風条件が多い

**結論:** 逃げ予測が有効な環境。現在の1コース重視モデルが機能。

### 低精度会場の失敗パターン（56-60%）
**会場:** 平和島(56.7%), 桐生(56.3%), 鳴門(60.4%)

❌ **共通特性:**
1. 1コース実績勝率が想定より低い (37-47% vs 予想 45-55%)
2. **非1コース決着が3割以上 (30-32%)** - モデルが予測できない
3. 不的中レースでの最多決まり手は「まくり」(43-58%)
4. 「まくり差し」の的中率が特に低い (0-20%)
5. 強風時の精度低下が顕著

**結論:** モデルが外郭艇の台頭を捉えられていない。まくり/まくり差しの予測強化が必須。

## 🔧 改善施策（3段階）

### Phase 1: 緊急キャリブレーション（1-2週間）
**難度:** ⭐ 低 | **効果:** ⭐⭐⭐ 中程度 | **実装:** コード修正のみ

> ⚠️ **重要な前提（model-auditor の calibration-report.js 検証で確定）**
> 「逃げパターンが多すぎる（patterns[0] が 977/1000 で nige）」ことと「逃げの確率値が低すぎる（distribution.nige 平均40.5% に対し実測53.1%、乖離14pt）」は **独立した別問題** である。
> さらに逃げの過小評価は週次で悪化中（0-40%帯の予測-実現が W15 -6.6pt → W20 -18.8pt）。
> したがって Phase 1 では **逃げ確率を一律で下げる施策（softmax温度低下・upsetFactor強化・まくりフロア一律引き上げ）を採用しない**。これらは逃げの過小評価をさらに悪化させる。
> Phase 1 は「逃げ確率を実測へ寄せる補正」を最優先とし、まくり系の相対的な押し上げは softmax 一律変更ではなく Phase 2 の展示タイム統合（rank 2）で実現する。

#### Phase 1a: 逃げ確率のキャリブレーション補正【最優先・時間依存】

逃げ rawProb のキャリブレーションスケーリングを追加し、distribution.nige の予測値を実測（約53%）へ寄せる。SOFTMAX_TEMP（turnPrediction.js:29）は逃げを下げる方向のため一律変更しない。必要なら softmax 全体を見直し、差し・まくり・抜きが高確率帯で逆に過信されている問題（抜き20-30%帯は実5.4%で乖離16.3pt）も併せて補正する。

**緊急性:** 逃げの過小評価はデータ蓄積では自己修正されず週次で拡大している。放置すると悪化し続けるため、他施策より先に着手する。

**検証:** calibration-report スキル（`scripts/analysis/calibration-report.js`）を修正前後で実行し、各確率帯の乖離縮小・ブライアスコア改善・週次悪化トレンドの停止を確認。

#### Phase 1b: 強風時の過剰抑制の是正

upsetFactor の風速・波高による逃げ抑制（turnPrediction.js:311-317、最大25%）が実測効果の約5倍（風速7m+でも逃げ率低下は0-2m比 -4.8pt のみ）。**強化ではなく縮小**し、windSpeed/waveHeight 係数を実測効果量に合わせ1/3〜1/5へ下げる。これも逃げの過小評価を緩和する方向に働く。

**検証:** turn-prediction-venue-deep-analysis.js の hit_rate_by_wind を修正前後で比較。

**予想効果:** 72.5% → 74-76% (+1.5-3.5pt)。加えて悪化トレンドの停止そのものが精度劣化の防止になる。

### Phase 2: 会場別モデル分化（2-3週間）
**難度:** ⭐⭐ 中 | **効果:** ⭐⭐⭐⭐ 大 | **実装:** 会場分類 + パラメータ分化

1. **会場を3グループに分類**
   ```javascript
   // venue_type 分類
   const VENUE_TYPE = {
     'stable': ['18', '19', '06', ...],    // 1コース勝率 55%+ （下関、徳山、浜名湖等）
     'volatile': ['01', '04', '14', ...],   // 1コース勝率 45-55% （桐生、平和島、鳴門等）
     'extreme': [],                         // 1コース勝率 45%未満 （該当なし）
   };
   ```

2. **グループ別 upsetFactor 倍率**
   ```javascript
   const upsetFactorMultiplier = {
     'stable': 1.0,      // そのままのupsetFactor
     'volatile': 1.5,    // 1.5倍に強化
     'extreme': 2.0,     // 2倍に強化
   };
   
   function calcUpsetFactor(raceConditions) {
     // ... 既存ロジック ...
     const venueType = getVenueType(raceConditions.venueCode);
     return clamp(
       factor * upsetFactorMultiplier[venueType], 
       0, 1
     );
   }
   ```

3. **決まり手別フロア値の会場別調整**
   ```javascript
   // volatile会場ではまくり/まくり差しのフロア値を上げる
   const TECHNIQUE_FLOOR_BY_VENUE = {
     'stable': {
       makuri: 0.015,
       makurizashi: 0.012,
     },
     'volatile': {
       makuri: 0.06,      // +4倍
       makurizashi: 0.04,  // +3倍
     },
   };
   ```

4. **テスト→検証**
   - 会場別で精度比較（特に低精度会場の向上を確認）
   - False positive（外郭予測が増えて他の精度低下）を監視

**予想効果:** Phase 1後の74-76% → 76-78% (+2-3pt)

### Phase 3: 決まり手別予測モデルの強化（4-8週間）
**難度:** ⭐⭐⭐ 高 | **効果:** ⭐⭐⭐⭐⭐ 最大 | **実装:** トレーニングデータ拡張 + 新特性

1. **トレーニングデータの拡張**
   - 現在: 2026-03-10～05-17 (1000レース)
   - 目標: 2025-01-01～2026-05-17 (半年分、~12000レース)
   - 特に「まくり決着」「まくり差し決着」のサンプルを1000以上確保

2. **新特性の導入**
   ```javascript
   // 1着艇グレードによる確信度調整
   const firstLaneGradeConfidence = {
     'A1': 1.0,    // 高確信
     'A2': 0.85,
     'B1': 0.6,    // 低確信（的中率 63.9%）
     'B2': 0.5,
   };
   
   // グレード低い場合は upsetFactor を動的に上げる
   const gradeUpsetBoost = 1 - firstLaneGradeConfidence[firstLaneGrade];
   rawProb *= (1 - gradeUpsetBoost * 0.2);  // 最大20%の追加抑制
   ```

3. **決まり手別の複合的な予測強化**
   - **まくり**: 進入コース差 + 展示ST差 + 近3走の被攻撃率を組み合わせ
   - **まくり差し**: まくり条件 + 2コースの追い上げ能力を同時判定
   - **差し**: 1コースの防御力 + 2-3コースのST差で判定精度向上

4. **venue-specific patterns の学習**
   - 各会場で「このコース組み合わせ＋この決まり手」は頻出 vs 稀少
   - 例: 下関では外郭のまくり差しは極めて稀 (1/22) → 確率を自動調整
   - 例: 平和島では外郭のまくり差しが頻出 (8/26外れ) → 確率を上げる

5. **段階的ロールアウト**
   - Phase 3-1: グレード調整のみ（コード修正）
   - Phase 3-2: トレーニングデータ拡張 + 再学習
   - Phase 3-3: 新特性導入 + A/Bテスト

**予想効果:** Phase 2後の76-78% → 79-81% (+3-4pt)

## 📝 注記

1. **Decision accuracy vs Correctness accuracy**
   - 現在の測定: 「上位3つのいずれかが当たれば的中」という decision accuracy
   - 将来考慮: Ranked prediction accuracy（1位の正解確率、2位確率等）

2. **Out-of-sample検証の重要性**
   - 現在: 2026-03-10～05-17 の固定期間で検証
   - Phase 3後: 2026-05-17以降のホールドアウトテスト期間で実績確認必須

3. **会場別ルール実装への波及**
   - 予測精度向上により、会場別ルール（単勝・複勝）の信頼度が向上
   - 既存ルールの回収率が自動的に改善される可能性
