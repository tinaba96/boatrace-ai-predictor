# i18n 需要計測レポート 運用ガイド

多言語化プロジェクト Phase 4（BOA-128）の計測基盤。英語版（/en/*）の実需要を測り、次言語（中国語・韓国語・東南アジア言語）への投資判断材料を作る。

## 計測の仕組み

| 計測項目 | 実装 | GA4 での見え方 |
|---------|------|--------------|
| 言語別ページビュー | `/en/*` パスで自動分離 + SPA ルート変更の `trackPageView` | ページパスで `/en` フィルタ |
| 表示言語 | `trackLanguage()` がユーザープロパティ `app_language` を送信 | カスタムディメンション（要登録） |
| 言語切替 | `trackLanguageSwitch()` が `language_change` イベントを送信 | イベント一覧 |
| 国別トラフィック | GA4 標準の地域レポート | ユーザー属性 > 地域 |

## 初回セットアップ

### 1. GA4 カスタムディメンションの登録（任意・推奨）

GA4 管理 > データの表示 > カスタム定義 > カスタムディメンションを作成:

- ディメンション名: `App Language`
- 範囲: ユーザー
- ユーザープロパティ: `app_language`

### 2. レポートスクリプト用の権限設定

1. GA4 管理 > プロパティ設定 > プロパティID（数字）を確認
2. `.env.local` に追加:
   ```
   GA4_PROPERTY_ID=123456789
   ```
3. GA4 管理 > プロパティのアクセス管理 > `GOOGLE_SERVICE_ACCOUNT_EMAIL` のアドレスを「閲覧者」として追加

## 月次レポートの実行

```bash
node scripts/analysis/i18n-demand-report.js           # 直近30日
node scripts/analysis/i18n-demand-report.js --days=90 # 直近90日
```

出力:
- コンソール: 言語別PV・英語ページの国別ユーザー・流入チャネル・言語切替回数
- JSON: `data/analysis/i18n-demand/report-YYYY-MM-DD.json`（推移比較用）

## 判断基準（3ヶ月後を目安）

deep-research 調査（2026-07）の結論「国際需要は未検証」に答えを出す:

| 指標 | 見るポイント |
|------|------------|
| 英語PVシェア | 全体の数%を継続的に超えるか |
| 国別分布 | 中国語圏 / 韓国語圏 / 東南アジアのどこが多いか → 次言語の優先度 |
| 流入チャネル | Organic Search が伸びているか（/en の SEO が効いているか） |
| 言語切替回数 | 日本語ページに来た外国人が EN に切り替えているか |
| 滞在・リピート | GA4 の標準レポートでエンゲージメント率を確認 |

次言語に進む場合は `src/i18n.js` の `SUPPORTED_LANGUAGES` に追加し、`src/locales/{lng}/common.json` を作成する（アーキテクチャは対応済み）。
