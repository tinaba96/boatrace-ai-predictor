開発要求：ボートレース予測サイト「BoatAI」機能拡張1. プロジェクトの全体方針コンセプト: 競合（ボートレース日和）のような大量のデータ提示ではなく、AIによる「結論（買い目）」と「視覚的な直感性」に特化したタイパ重視のサイト。ビジネスモデル: ユーザー登録・課金なし。将来的に公営競技（施行者）の広告収益をモデルとするため、クリーンで新規ファンにも分かりやすいUI/UXを目指す。技術制約: LLMのAPIコストを最小化するため、計算ロジックやアニメーション制御はフロントエンド（React/TS）で完結させる。2. 追加実装機能の詳細A. 1マーク展開予測アニメーション目的: AIが予測した「決まり手」を視覚化し、ユーザーの納得感を高める。仕様:描画: SVG + framer-motion を使用。パターン: nige (1号艇逃げ), sashi (内差し), makuri (外から叩く), makuri-sashi (間を割る) の4種をPropsで切り替え。演出: 旋回時のスピード変化（イージング）と、ボートの向き（rotation）の制御。状態管理: isAnalyzing ステートを用いた「AI分析中」のスケルトン演出。B. 期待値（Expected Value）計算・表示エンジン目的: 「当たる確率」だけでなく「オッズを含めた美味しさ」を提示する。計算ロジック: * 期待値 = AI的中確率(%) × 現在オッズ※計算はLLMではなくJavaScriptの関数で行う。表示: * 期待値が1.0を超える買い目に「お宝/期待値高」のラベルを表示。期待値ランク（S, A, B, C）による視認性の向上。C. スマート資金配分シミュレーター目的: ユーザーが予算を入力するだけで、合成オッズに基づいた最適な購入金額を算出する。仕様:ユーザーが買い目を選択し、合計予算を入力。合成オッズ（$1 / \sum(1/Odds_n)$）を算出し、各買い目に配分額を自動割り振り。ユーザー登録不要: ブラウザの LocalStorage を利用して、直近の予算設定などを保持する。
