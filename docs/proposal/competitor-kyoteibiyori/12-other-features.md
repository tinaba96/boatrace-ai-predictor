# その他機能の詳細分析

## フライング休み表示

### 概要

選手のフライング（F）ペナルティに伴う出場停止期間を確認できる機能。「隠れフライング」（出走表ではF0表記だが未消化のフライングがある状態）の検出にも対応している。

### 機能詳細

| 機能 | 説明 | アクセス方法 |
|------|------|-------------|
| F休み期間一覧表 | 現在フライング休み中の全選手リスト | データ分析 > F休み期間タブ |
| 隠れフライング表示 | 未消化フライングがある選手を表示 | 出走表 > 基本情報 > フライング欄 |
| F休み開始日 | 選手情報にフライング休み開始日を表示 | 選手詳細ページ |

### 隠れフライングの表示方法

- 出走表の基本情報欄に「フライング」項目がある
- 「未消化」の欄に数字が表示されている場合、隠れフライング状態
- 赤枠で「1」と表示される選手が隠れフライング

### 追加日

- F休み期間一覧: 2022年11月11日追加
- 隠れフライング検出: 2020年11月28日追加

### 予想への影響

隠れフライングの選手はスタートが慎重になる傾向があり、逃げ・まくりが決まりにくくなる。この情報は予想の重要な判断材料。

## 売上履歴（日別/節別/場別/全体）

### 概要

2024年1月13日に追加された機能で、各場の売上データを多角的に分析できる。

### 確認可能なデータ

| 集計軸 | 説明 |
|--------|------|
| 日別売上 | 特定日の売上金額 |
| レース別売上 | レース単位の売上金額 |
| 節別売上 | 開催節ごとの売上合計 |
| 年間売上 | 年間の売上推移 |
| 場別売上 | 場ごとの売上比較 |

### 注意事項

- 一部の場（戸田、江戸川、平和島、多摩川）はレース別売上が未対応
- 今後オッズ等との組み合わせ分析が予定されている

### アクセス方法

- データ分析 > 平均値データ・売上履歴

## 発売票数履歴・リアルタイム売上

### 概要

2024年6月1日に追加された機能。締切前のオッズ変動だけでなく、実際の販売枚数（票数）の推移も追跡できる。

### 発売票数履歴

| 項目 | 説明 |
|------|------|
| アクセス方法 | 出走表 > オッズ一覧タブ > 発売票数履歴 |
| 表示内容 | 各投票式（3連単、3連複等）の時間帯別発売枚数 |
| 活用方法 | オッズだけでなく実際の売上規模を確認し、「厚く買われている買い目」を把握 |

### リアルタイム売上

| 項目 | 説明 |
|------|------|
| アクセス方法 | トップページ > 全国の売上状況 |
| 表示内容 | 当日の各場のリアルタイム売上金額 |
| 活用方法 | 売上が多い場は注目度が高く、オッズが安定しやすい傾向 |

## 舟券メモ機能

### 概要

購入した舟券を記録し、選手ごとの購入履歴や回収率を分析できる会員専用機能。ユーザー登録（無料）が必要。

### 舟券の登録方法

1. 出走表画面で「オッズ一覧」タブを選択
2. 「舟券メモ」ボタンを押す
3. メモしたい買い目をチェックして検索
4. 登録候補から金額を入力して登録ボタンを押す

### 確認できるデータ

| データ | 説明 |
|--------|------|
| 選手ごとの購入舟券 | 特定選手が出走したレースで購入した舟券一覧 |
| 回収率 | 購入金額に対する払戻金額の割合 |
| 的中率 | 購入した舟券の的中率 |
| 出目履歴 | 過去の購入出目パターン |
| 該当艇メモ履歴 | 特定の艇番での購入履歴 |

### 履歴の確認方法

**方法1: 出走表から**
- 舟券メモを確認したい選手の「履歴」ボタンを押す
- 該当艇での出目履歴と的中率、回収率を表示
- 全ての艇での出目履歴と的中率、回収率も表示可能

**方法2: 選手一覧から**
- 選手一覧で特定選手を検索
- 「舟券メモ履歴」ボタンで該当選手のメモ舟券一覧を表示
- 艇番指定で絞り込み可能

## 回収率分析

### 概要

舟券メモに記録されたデータを基に、自分の舟券購入パターンの成績を多角的に分析する機能。

### 分析の切り口

| 分析軸 | 説明 |
|--------|------|
| 期間指定 | 日付で期間を指定して集計 |
| 場別指定 | 特定の場に絞って集計 |
| 表示方法:組合せ | 買い目の組み合わせ別に回収率を表示 |
| 表示方法:選手別 | 選手別に回収率を表示 |

### アクセス方法

ユーザー情報 > 舟券メモ分析タブ

### 活用例

- 「平和島で買った舟券の回収率が特に低い」→ 場の得意不得意を把握
- 「4号艇の選手を絡めた3連単の的中率が高い」→ 自分の予想パターンの強みを発見
- 「直近3ヶ月の回収率が下降傾向」→ 予想方法の見直しの判断材料

## まとめニュース（日刊スポーツ連携）

### 概要

スポーツ紙のボートレース関連ニュースをまとめて表示する機能。

URL: `https://kyoteibiyori.com/news/`

### 配信元

| 配信元 | 割合 |
|--------|------|
| 日刊スポーツ | 大多数の記事 |
| デイリースポーツ | 一部記事 |

### 表示形式

| 項目 | 説明 |
|------|------|
| サムネイル画像 | 記事の代表画像 |
| タイトル | 記事のタイトル |
| 配信元ロゴ | 配信元の識別 |
| 配信日時 | 年月日と時刻 |
| 外部リンク | 元記事への遷移リンク |

### 特徴

- 時系列順で30件以上を一覧表示
- カテゴリ分けはなく、全ニュースをフラットに表示
- レース結果、選手コメント、グレード別レース情報、事件/アクシデント等を網羅
- 前日のニュースも遡って確認可能

## BoatAIへの実装示唆

### 優先度: 高

1. **舟券メモ + 回収率分析**: BoatAIの予想に基づいて購入した舟券の成績を自動追跡する機能。BoatAIの予想精度の「証明」にもなるため、ユーザーの信頼獲得に直結する。ボートレース日和は手動入力だが、BoatAIは予想連動で半自動化できる可能性がある

2. **フライング休み/隠れフライング表示**: 予想ロジックにも直結する重要情報。出走表に「隠れF」マークを表示するだけでも差別化になる

### 優先度: 中

3. **ニュース集約**: スポーツ紙のボートレース記事をまとめて表示。ユーザーの滞在時間向上とSEO効果が期待できるが、配信元との契約が必要

4. **売上データ表示**: 場別の売上推移を表示し、「人気の場」「注目レース」を可視化。BoatAIの予想対象選定にも活用可能

5. **MyData的なカスタマイズ**: ユーザーが見たい情報を選べるカスタマイズ機能。ただしBoatAIはAI予想が主軸なので、「AIが判断材料として重視した項目」をハイライト表示する方がユーザー体験として優れている

### 優先度: 低

6. **ブログ/コラム強化**: データ分析に基づくオリジナルコンテンツの拡充。既に `/public/blog/` で実施中

7. **動画コンテンツ**: YouTube連携。コンテンツ制作コストが高いため、優先度は低い

8. **番組マンの癖解析**: ユニークな機能だがデータ取得・分析の難易度が高い

### 差別化ポイント

- ボートレース日和は「300〜400項目のデータを網羅する百科事典型」のサービス
- BoatAIは「AIが最適な項目を自動選定し、予想として提示するエキスパート型」のサービス
- 舟券メモ機能については「AI予想との連動」で大きく差別化可能。例えば「AIルールAの推奨レースのみの回収率」といった切り口はボートレース日和にはない
- フライング休み表示は実装コストが低く効果が高い。出走表取得時にフライングデータも併せて取得し、予想ロジックにも組み込むのが理想
