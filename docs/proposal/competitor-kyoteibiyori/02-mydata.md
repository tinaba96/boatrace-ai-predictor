# MyData（カスタマイズ出走表）の詳細分析

## 概要

MyDataは、ボートレース日和（kyoteibiyori.com）が提供するカスタマイズ出走表機能。約400項目のデータの中から、ユーザーが自分の予想スタイルに合ったデータ項目を選択し、オリジナルの出走表を作成できる。無料会員登録のみで利用可能。

### 機能の位置づけ

- ボートレース日和の最大の差別化ポイント
- 出走表の9つのタブの1つ（slider=8）として表示
- 300〜400項目から自由に選択・並び替えが可能
- 多くの解説サイトで「まずMyDataを設定すべき」と推奨されている

### 利用条件

- 無料会員登録（メールアドレス + パスワード）が必要
- 登録後すぐに利用可能
- 設定はマイページから行う

---

## 設定フロー（ステップバイステップ）

### Step 1: 会員登録

1. ボートレース日和トップページにアクセス
2. 会員登録ページでメールアドレスとパスワードを入力
3. 登録完了

### Step 2: ログイン

1. メールアドレスとパスワードを入力
2. 「ログイン」をクリック

### Step 3: MyData設定画面へ移動

1. マイページを開く
2. 「MyData」タブをクリック

### Step 4: データ項目の選択

1. 「下記から登録するデータを選択」セクションを表示
2. カテゴリを選択（基本、枠別、決まり手等）
3. 各項目の「追加」ボタンを押して、表示したい項目を追加

### Step 5: 表示順の並び替え

1. 登録済みデータの一覧で、順番を変更したい項目の「上へ」ボタンを押す
2. 上方向にのみ移動可能（下に戻す場合は削除→再追加）

### Step 6: 登録の確定

1. すべての項目の選択・並び替えが完了したら「登録」ボタンをクリック
2. 「登録しました」と確認メッセージが表示される

**注意点**:
- 「追加」ボタンを押しただけでは登録されない。最後に必ず「登録」ボタンを押すこと
- 下方向への並び替えはできないため、順番を変更する場合は削除→再追加が必要
- データの追加・削除後に登録ボタンを押すことで反映される

### Step 7: 出走表で確認

1. 出走表ページを開く
2. データ変更タブを右端までスクロール
3. 「MyData」タブ（slider=8）を選択
4. 設定した項目がカスタム出走表として表示される

## 選択可能なデータカテゴリと項目（全カテゴリを網羅）

約400項目のデータは、以下のカテゴリに分類される。各カテゴリ内でさらに期間や条件で細分化され、膨大な選択肢を構成している。

### カテゴリ1: 基本データ

選手の基本的な成績情報。

| 項目名 | 説明 | 期間バリエーション |
|--------|------|-------------------|
| 勝率 | 全国通算勝率 | 通算 / 直近6ヶ月 / 直近3ヶ月 / 直近1ヶ月 |
| 2連対率（2連率） | 2着以内に入る確率 | 通算 / 直近6ヶ月 / 直近3ヶ月 |
| 3連対率（3連率） | 3着以内に入る確率 | 通算 / 直近6ヶ月 / 直近3ヶ月 |
| 1着率 | 1着になる確率 | 通算 / 直近6ヶ月 |
| 平均ST | 平均スタートタイミング | 通算 / 直近6ヶ月 / 直近3ヶ月 |
| 級別 | A1/A2/B1/B2 | 現期 |
| 事故率 | 事故の発生率 | 現期 |

**条件別成績（各指標とも勝率・2連率・3連率を含む）**:
- ナイターレース限定
- SG・G1競走限定
- 女子戦限定

|--------|------|-------------------|

| 枠別2連対率 | 該当コースでの2連対率 | 通算 / 直近1年 / 直近6ヶ月 |
| 枠別3連対率 | 該当コースでの3連対率 | 通算 / 直近1年 / 直近6ヶ月 |
| 枠別出走回数 | 該当コースでの出走回数 | 通算 / 直近1年 |
| 枠別勝率 | 該当コースでの勝率 | 通算 / 直近1年 / 直近6ヶ月 |
| 枠別平均ST | 該当コースでの平均ST | 直近1年 / 直近6ヶ月 |
| 枠別ST順位 | 6艇中のST順位 | - |

**条件別枠別成績**:
- 総合

選手の勝ちパターン（決まり手）を数値化。各コースについて取得可能。

## おすすめ設定例（複数パターン）

### パターン1: 初心者向け（基本4項目）

最小限の項目で予想の基本を押さえる設定。

| 項目名 | 選択理由 |
|--------|---------|
| 1着率 | 最もシンプルな実力指標 |
| 2連対率 | 舟券の軸選びに直結 |
| 3連対率 | 3連系舟券の参考に |
| 平均ST（直近6ヶ月） | スタート力の判断に必須 |

### パターン2: 中級者向け（予想師推奨設定）

枠別情報と決まり手を加えた実践的な設定。

| 項目名 | 選択理由 |
|--------|---------|
| 平均ST（直近6ヶ月） | 直近のスタート傾向 |
| 勝率（直近6ヶ月） | 直近の調子を反映 |
| 枠別1着率 | コース適性の判断 |
| 枠別2連対率 | コース別の連対力 |
| **枠別 決まり手（総合） 上:直近1年 / 下:直近6ヶ月** | **マスト項目** -- レーススタイルの把握 |
| **枠別 決まり手数（総合） 直近1年** | **マスト項目** -- 決まり手のサンプル数確認 |
| モーター勝率（直近1ヶ月） | 直近のモーター状態 |
| 今節展示順位 | 当日の機力判断 |

**予想師が「マストで入れるべき」と推奨する独自データ**:
- 「枠別 決まり手（総合） 上:直近1年 / 下:直近6ヶ月」
- 「枠別 決まり手数（総合） 直近1年」

この2項目により、各選手が該当コースでどのような勝ち方をするかの傾向と、そのサンプル数を同時に確認できる。

### パターン3: 展開予想重視型

## 技術的な実装ポイント

### データ構造

```
MyData設定 = {
  userId: ユーザーID,
  items: [
    {
      itemId: データ項目ID,
      category: カテゴリ名,
      displayOrder: 表示順序（整数）
    },
    ...
  ]
}
```

### URL設計

- 出走表のMyDataタブは `slider=8` パラメータで管理
- 例: `race_shusso.php?place_no=14&race_no=1&hiduke=20240703&slider=8`

### データ取得の仕組み（推定）

1. ユーザーのMyData設定をサーバーサイドで保持（DB or セッション）
2. 出走表ページのMyDataタブ表示時に、設定に基づいてデータを動的に生成
3. 各データ項目は事前計算済みのテーブルから取得（リアルタイム計算ではない）

### 項目の管理方式（推定）

- 約400項目は、カテゴリ + サブカテゴリ + 期間 の組み合わせで構成
- 例: 「枠別」+「1着率」+「直近6ヶ月」= 1項目
- 項目IDをキーにして、表示順序をユーザーごとに管理

### フロントエンド実装の特徴

- タブの横スクロール（CSS `overflow-x: scroll` or swipe ライブラリ）
- データテーブルは選手6名 x 選択項目N行の形式
- 項目が多い場合は縦スクロールで対応
- 「上へ」ボタンによる並び替えはシンプルだが、下方向移動ができない制約あり

## BoatAIへの実装示唆

### 短期的に取り入れるべき要素

#### 1. 出走表へのデータ項目追加

現在のBoatAIの出走表は最小限の情報しか表示していない。以下のデータを追加するだけで大幅に改善可能:

- **枠別1着率・2連対率** -- ルールエンジンで既に使用しているデータのユーザー向け可視化
- **平均ST（直近6ヶ月）** -- スタート力の表示
- **モーター2連対率** -- 機力の表示

これらはボートレース日和でも「基本中の基本」として位置づけられている。

#### 2. 決まり手データの表示

BoatAIの強みであるルール分析と直結する項目:

- **枠別決まり手率** -- 「この選手は2コースでまくりが多い」等の情報
- **逃し率（2コース）** -- インが逃げるかの判断材料

これらは予測の「根拠説明」としても活用でき、ユーザーの納得感を高められる。

#### 3. フライング情報の表示

フライング持ちの選手はスタートを控えめにする傾向があり、予想に大きく影響する:

- **フライング未消化数** -- 簡単に取得可能なデータで、予想精度に直結

### 中期的に検討すべき要素

#### 4. 簡易カスタマイズ機能

MyDataのような400項目のフルカスタマイズは過剰だが、以下のような簡易版は実装価値あり:

- **プリセット切り替え**: 「基本」「詳細」「展開重視」の3パターンから選択
- **項目の表示/非表示トグル**: 各データ項目の表示・非表示を切り替え
- **ローカルストレージでの保存**: 会員登録不要でブラウザに設定を保存

#### 5. AI予測の根拠表示

BoatAI独自の強みとして、MyDataタブの代わりに「AI分析」タブを設置:

- ルールマッチングの結果（どのルールに該当したか）
- 信頼度スコアの根拠（どのデータが決め手になったか）
- 過去の類似レースでの的中率

#### 6. 直前データの統合

展示データをスクレイピングして予測に反映:

- 展示タイム差の表示
- 展示STと過去の平均STの比較
- 部品交換の有無

### 長期的なビジョン

#### 7. データプラットフォーム化

将来的には、ボートレース日和のような「データ提供」とBoatAIの「AI予測」を融合:

- ユーザーが重視するデータ項目を学習し、パーソナライズされた予測を提供
- 過去の購入履歴と照合して、ユーザーごとの最適な表示項目を自動提案
- API提供による外部連携（予想ツール開発者向け）

| 優先度 | 項目 | 工数目安 | 効果 |

| P0 | 枠別1着率・2連対率の表示追加 | 小 | 高 |
